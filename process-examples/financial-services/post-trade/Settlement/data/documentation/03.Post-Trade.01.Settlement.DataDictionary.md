## Process Input Data

### Core Settlement Fields

#### settlementId

- **Type:** String
- **Format:** `SETTLE-YYYYMMDD-XXXXXX`
- **Required:** Yes
- **Description:** Unique identifier for the settlement instruction.
- **Example:** `SETTLE-20251014-S1T2L3`

#### tradeId

- **Type:** String
- **Format:** `TRADE-YYYYMMDD-XXXXXX`
- **Required:** Yes
- **Description:** The ID of the trade to be settled.

#### instrument

- **Type:** Object
- **Required:** Yes
- **Description:** Details of the financial instrument being settled.
- **Properties:**
  - `instrumentId`: (String) Unique identifier for the instrument (e.g., ISIN, CUSIP).
  - `instrumentType`: (Enum) Type of instrument (e.g., `equity`, `bond`, `derivative`).

#### quantity

- **Type:** Number
- **Required:** Yes
- **Description:** The quantity of the instrument to be settled.

#### price

- **Type:** Number
- **Required:** Yes
- **Description:** The settlement price.

#### currency

- **Type:** String
- **Format:** ISO 4217 currency code
- **Required:** Yes
- **Description:** The settlement currency.

#### settlementDate

- **Type:** Date
- **Format:** ISO 8601 date (YYYY-MM-DD)
- **Required:** Yes
- **Description:** The intended settlement date.

#### counterparties

- **Type:** Array of Objects
- **Required:** Yes
- **Description:** The counterparties involved in the settlement.
- **Object Properties:**
  - `counterpartyId`: (String) The ID of the counterparty.
  - `settlementInstructions`: (Object) The settlement instructions for the counterparty.

## Task Output Data

### Enrich Settlement Instructions Output

#### enrichedInstructions

- **Type:** Object
- **Required:** Yes
- **Description:** The enriched settlement instructions.

### Generate Settlement Instruction Message Output

#### instructionMessage

- **Type:** String
- **Required:** Yes
- **Description:** The generated settlement instruction message (e.g., in ISO 20022 format).

### Monitor Settlement Status Output

#### settlementStatus

- **Type:** Enum
- **Values:** `Pending`, `Partial`, `Complete`, `Failed`
- **Required:** Yes
- **Description:** The current status of the settlement.

### Fail Resolution Output

#### resolution

- **Type:** Object
- **Required:** Yes
- **Description:** The resolution for a failed settlement.
- **Properties:**
  - `reason`: (String) The reason for the failure.
  - `resolutionAction`: (String) The action taken to resolve the failure.

## Reference Data

### Standing Settlement Instructions (SSIs)

#### ssiId

- **Type:** String
- **Description:** Unique identifier for the SSI.

#### counterpartyId

- **Type:** String
- **Description:** The ID of the counterparty the SSI belongs to.

#### settlementAgent

- **Type:** String
- **Description:** The settlement agent for the counterparty.
